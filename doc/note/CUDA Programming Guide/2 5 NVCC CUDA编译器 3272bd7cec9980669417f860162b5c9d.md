# 2.5 NVCC CUDA编译器

`nvcc` 是 NVIDIA 用于编译 CUDA C/C++及 PTX 代码的**工具链（Toolchain）**。

该工具链是 CUDA 工具包的一部分，包含**编译器**、**链接器**以及 **PTX 和 Cubin 汇编器**等多个工具。

本章介绍了构建应用程序时所需的 `nvcc` 最常见用法及细节。

---

# **CUDA 源文件与头文件**

使用 `nvcc` 编译的源文件可能包含主机代码和设备代码的组合，主机代码在 CPU 上执行，设备代码在 GPU 上执行：

- `nvcc` 接受常见的 C/C++**源文件**扩展名 `.c` 、 `.cpp` 、 `.cc` 、 `.cxx` 用于纯主机代码，以及 `.cu` 用于包含**设备代码或混合主机与设备代码**的文件。
- 包含设备代码的**头文件**通常采用 `.cuh` 扩展名，以区别于**纯主机代码头文件** `.h` 、 `.hpp` 、  `.hh`、 `.hxx` 等。

| 文件扩展名 | 描述 | 内容 |
| --- | --- | --- |
| `.c` | C 源文件 | 主机端代码 |
| `.cpp`, `.cc`, `.cxx` | C++ 源文件 | 主机端代码 |
| `.h`, `.hpp`, `.hh`, `.hxx` | C/C++头文件 | 设备端代码、主机端代码、主机/设备端混合代码 |
| `.cu` | CUDA 源文件 | 设备端代码、主机端代码、主机/设备端混合代码 |
| `.cuh` | CUDA 头文件 | 设备端代码、主机端代码、主机/设备端混合代码 |

# NVCC 编译工作流

1. 在初始阶段， `nvcc` 会将设备代码与主机代码分离，并分别分派给 GPU 编译器和主机编译器进行编译。

1. 仅包含主机代码的文件可以通过 `nvcc` 或直接使用主机编译器进行构建。
2. 生成的目标文件可在**链接**时与来自 `nvcc`的包含 GPU 代码的目标文件合并。

1. GPU 编译器将 C/C++ 设备代码编译为 **PTX 汇编代码**。GPU 编译器会针对编译命令行中指定的每个虚拟指令集架构（例如 `compute_90` ）分别执行编译。
2. 单独的 PTX 代码随后会传递给 `ptxas` 工具，该工具针对目标硬件指令集架构（由其 SM 版本标识）生成 **Cubin**。
3. (可选的) 可以将多个 PTX 和 Cubin 目标嵌入到**应用程序**或库中的单个二进制**Fatbin**容器内，以支持多个虚拟及目标硬件指令集架构。

上述工具的调用与协调工作均由 `nvcc` 自动完成。

可使用 `-v` 选项显示完整的编译流程与工具调用情况。

使用 `-keep` 选项可以将编译过程中生成的中间文件保存在当前目录或 `--keep-dir` 指定的目录中。

---

针对 CUDA 源文件 `example.cu` 的编译流程：

```cpp
// ----- example.cu -----
#include <stdio.h>
__global__ void kernel() {
    printf("Hello from kernel\n");
}

void kernel_launcher() {
    kernel<<<1, 1>>>();
    cudaDeviceSynchronize();
}

int main() {
    kernel_launcher();
    return 0;
}
```

# **NVCC 基础用法**

使用 `nvcc` 编译 CUDA 源文件的基本命令为：

```bash
nvcc <source_file>.cu -o <output_file>
```

`nvcc` 接受常见的编译器标志，用于指定包含目录 `-I <path>` 和库路径 `-L <path>` 、链接其他库 `-l<library>` 以及定义宏 `-D<macro>=<value>` 。

```bash

nvcc example.cu -I path_to_include/ -L path_to_library/ -l cublas -o <output_file>

```

## **NVCC PTX 与 Cubin 生成**

**默认情况**下， `nvcc` 会为 CUDA 工具包所支持的**最早 GPU 架构**（最低 `compute_XY` 和 `sm_XY` 版本）生成 PTX 和 Cubin 文件，以**最大化兼容性**。

- `-arch` 选项可用于为特定 GPU 架构生成 PTX 和 Cubin 文件。
- `-gencode` 选项可用于为多个 GPU 架构生成 PTX 和 Cubin 文件。

```bash
nvcc example.cu -arch=compute_<XY> # e.g. -arch=compute_80 for NVIDIA Ampere GPUs and later
                                   # PTX-only, GPU forward compatible

nvcc example.cu -arch=sm_<XY>      # e.g. -arch=sm_80 for NVIDIA Ampere GPUs and later
                                   # PTX and Cubin, GPU forward compatible

nvcc example.cu -arch=native       # automatically detects and generates Cubin for the current GPU
                                   # no PTX, no GPU forward compatibility

nvcc example.cu -arch=all          # generate Cubin for all supported GPU architectures
                                   # also includes the latest PTX for GPU forward compatibility

nvcc example.cu -arch=all-major    # generate Cubin for all major supported GPU architectures, e.g. sm_80, sm_90,
                                   # also includes the latest PTX for GPU forward compatibility
```

## **主机代码编译注意事项**

**不包含设备代码或符号的编译单元**（即源文件及其头文件）可以直接使用**主机编译器**进行编译。

如果任何编译单元使用了 CUDA 运行时 API 函数，则应用程序必须与 **CUDA 运行时库链接**。（**提供静态库`libcudart_static`**和**共享库`libcudart`**）

nvcc**默认链接**静态CUDA运行时库。要使用共享库版本需要向nvcc传递 `--cudart=shared`标志。

---

`nvcc` 允许通过 `-ccbin <compiler>` **参数指定**用于主机函数的主机编译器。

也可以**定义环境变量** `NVCC_CCBIN` 来指定 `nvcc` 使用的主机编译器。

通过向 `nvcc` 传递 `-Xcompiler` 参数，可以将**参数透传**给主机编译器。

```bash
nvcc example.cu -ccbin=clang++

export NVCC_CCBIN='gcc'
nvcc example.cu -Xcompiler=-O3
```

## **GPU 代码的单独编译**

`nvcc` 默认采用**全程序编译模式**，该模式要求所有 GPU 代码和符号在使用它们的**编译单元中都存在**。

**分别编译模式（seperate compilation）**可以调用**其他编译单元**中定义的设备函数或访问设备变量，但必须在 `nvcc` 命令行中指定 `-rdc=true` 或其别名标志 `-dc` 

- 分别编译支持**更灵活的代码组织方式**，有助于**缩短编译时间**，并能生成**更小的二进制文件**。
- 与全程序编译相比，单独编译可能会引入一定的构建复杂度。由于设备代码链接可能影响性能，因此默认未启用该功能。

---

**分别编译**需满足以下条件：

- 在一个编译单元中定义的**非 `const` 设备变量**，在其他编译单元中必须使用 `extern` 关键字进行引用。
- 所有 **`const` 设备变量**必须使用 `extern` 关键字进行定义和引用。
- 所有 CUDA 源文件 `.cu` 必须使用 `-dc` 或 `-rdc=true` 标志进行编译。

**主机函数**和**设备函数**默认具有外部链接，且无需使用 `extern` 关键字。

分别编译 `definition.cu`, `example.cu`示例：

```cpp
// ----- definition.cu -----
extern __device__ int device_variable = 5;
__device__ int device_function() { return 10; }
```

```cpp
// ----- example.cu -----
extern __device__ int device_variable;
__device__ int device_funcition()

__global__ void kernel(int* ptr)
{
		device_variable = 0;
		*ptr = device_function();
}
```

```bash
nvcc -dc definition.cu -o definition.o
nvcc -dc example.cu -o example.o
nvcc difinition.o example.o -o program
```
