# 4.1 统一内存

本节将详细解释每种统一内存范式的具体行为和使用方式。

# 完全支持CUDA统一内存的设备上的统一内存

这些系统包括**硬件一致性内存系统**，例如 NVIDIA Grace Hopper 

启用**异构内存管理 (HMM)** 的**现代 Linux 系统**。HMM 是一种**基于软件的内存管理系统**，提供与硬件一致性内存系统相同的**编程模型**。

<aside>
💡

**硬件一致性系统：**将 CPU 和 GPU 共用统一页表的系统

**软件一致性系统：**CPU 与 GPU 使用独立页表的系统

</aside>

## 深入示例

完全支持 CUDA 统一内存的系统允许**设备访问**与该**设备交互的主机进程所拥有的任何内存**。

本节展示几个高级用例，所采用的内核仅将输入字符数组的前 8 个字符打印到标准输出流中：

```cpp
__global__ void kernel(const char* type, const char* data)
{
		static const int n_char = 8;
		printf("%s - first %d characters: '", type, n_char);
		for (int i = 0; i < n_char; i++)
		{
				printf("%c ", data[i]);
		}
		printf("'\n");
}
```
